# TenderMate

SaaS-платформа для автоматизации участия в госзакупках и коммерческих
тендерах (44-ФЗ, 223-ФЗ). Соло-проект, фаза MVP, цель — быстрая валидация
на рынке. Читай `PROJECT_LOG.md` в этом репозитории целиком перед началом
работы — там хронология решений и почему они приняты именно так.

## Главный принцип

Запрещено строить продукт как `PDF → LLM → Summary`. Правильная схема:

```
Tender Documents → Requirement Extraction (по документу отдельно)
→ Structured Requirements (факты с источником/пунктом)
→ Company Knowledge Base (профиль компании, статус МСП)
→ Compliance Engine → Risk Engine → Application Builder
```

LLM — вспомогательный инструмент, не ядро системы. Таблицы спецификации
(характеристики/цена/количество) парсятся БЕЗ LLM — прямым структурным
парсингом (`spec_table.py`). Причины: экономия токенов, обычные
формулировки состава товара ложно триггерят фильтры безопасности LLM,
выше точность (нет риска галлюцинации на длинных списках).

## Explainability — жёсткое правило

Каждый вывод системы обязан содержать источник (пункт/раздел/страница).
Если источника нет — вывод недостоверен. Не позволяй модели придумывать
номера пунктов, которых нет среди извлечённых фактов.

## Стек и репозитории

- Frontend: React 18 + Vite + TypeScript + Tailwind → Vercel.
  Репозиторий приватный: `github.com/misiknordic-design/TenderMate`
- Backend: FastAPI + Python, async → Timeweb App Platform (РФ).
  Репозиторий публичный: `github.com/misiknordic-design/TenderMate-backend`
- DB: PostgreSQL на Timeweb (`asyncpg`, не ORM). Приватная сеть между
  App Platform и БД не работает — подтверждённый баг Timeweb для уже
  существующих приложений. Сейчас: публичный IP у базы + TLS
  (`sslmode=verify-full`) + firewall на TCP/5432. Детали — в `PROJECT_LOG.md`.
- AI: Yandex Cloud AI Studio (OCR + LLM), модель через `YANDEX_MODEL_URI`.
  Anthropic НЕ используется в проде (гео-блокировка API с российских IP).
- Автозаполнение по ИНН: DaData API.

## Инфраструктурное правило (жёсткое)

Вся инфраструктура ТОЛЬКО РФ-совместимая. Разрешено: Timeweb Cloud,
Timeweb App Platform, Yandex Cloud, Yandex Cloud AI Studio, DaData,
Vercel (фронтенд — статика). Ничего иностранного в проде.

## Архитектурные ограничения MVP

Запрещено: Kubernetes, микросервисы, Kafka, Elasticsearch, Qdrant,
Redis Cluster. Вся система — на одном сервере. При выборе между простой
и сложной архитектурой — всегда простая (KISS/YAGNI).

Асинхронные задачи (OCR, обработка PDF, эмбеддинги, AI-анализ, генерация
документов) — ТОЛЬКО через FastAPI BackgroundTasks, никогда не в
HTTP-запросе напрямую.

## Правила кода

- Файлы — до 250 строк. Если правка превышает лимит — сокращай
  комментарии/докстринги, не логику.
- Не переписывай архитектуру без явной причины, не добавляй зависимости
  без необходимости.
- Перед нетривиальной правкой — одно предложение, почему это самый
  простой рабочий вариант.
- Точечные правки предпочтительны — редактируй файлы напрямую через diff,
  не пересобирай с нуля.
- Не логировать: пароли, JWT, email, телефоны, персональные данные.
  Секреты — только через переменные окружения, никогда в коде или в этом
  файле.
- **Бэкенд-репозиторий публичный** — не коммить туда ничего чувствительного
  (ключи, внутренние IP, пароли БД) даже во временных файлах или комментариях.

Фон `#F5F6FA`, карточки `#FFFFFF`, акцент малахит `#0F766E`, успех

Шрифты Golos Text (основной) + JetBrains Mono (числа). Логотип: «Tender»

копия конкурентов (semantor.ru, neirotender.ru, gos-tender-ai.ru,
tenderai.pro).

## Продуктовый принцип

Пользователь платит не за AI-анализ, а за: понимание требований,
выявление рисков, поиск подтверждающих документов, подготовку заявки.
Все решения должны увеличивать ценность именно этих сценариев.
TenderMate НЕ занимается ценовой стратегией участника («коридор для
торга» убран намеренно).
